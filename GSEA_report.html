Analysis Pipeline

This Gene Set Enrichment Analysis was performed using smartGSEA v0.1.0, implementing the GSEA algorithm (Subramanian et al., 2005) via clusterProfiler (Yu et al., 2012).

1. Data Preparation

  • Input: DESeq2 differential expression results
  • Contrast: M1 vs S1
  • Total genes: 16,060
  • Approach: All genes included (no pre-filtering on significance)

2. Gene ID Conversion

  • Method: clusterProfiler bitr() function
  • Conversion: ENSEMBL → ENTREZ gene IDs
  • Organism: human (auto-detected)
  • Success rate: 85.6%
  • Duplicate handling: Retained gene with highest |ranking metric|

3. Gene Ranking

  • Metric: log2fc
  • Sort order: Descending

4. GSEA Execution

  • Algorithm: Fast Gene Set Enrichment Analysis (fGSEA)
  • Gene sets: Gene Ontology Biological Process (GO:BP)
  • Permutations: 1,000
  • Gene set size: 10–500 genes

5. Multiple Testing Correction

  • Method: Benjamini-Hochberg FDR
  • Threshold: FDR < 0.05
  • Critical: Filtered on adjusted p-values (not nominal)
⚠️ Why This Matters
The default pvalueCutoff in clusterProfiler filters on nominal p-values, inflating results by 5–10×. smartGSEA enforces proper FDR filtering to ensure only statistically significant pathways are reported.
